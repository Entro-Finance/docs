# List Cards

Retrieve a paginated list of all cards associated with the authenticated account.

## GET /api/cards

List all cards with optional filtering and pagination.

<ParamField header="Authorization" type="string" required>
  Bearer authentication header of the form Bearer &lt;token&gt;, where &lt;token&gt; is your auth token.
</ParamField>

<ParamField query="page" type="string" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="string" default="25">
  Number of results per page (max 100)
</ParamField>

<ParamField query="status" type="string" optional>
  Filter by card status: `active`, `frozen`, `expired`, `cancelled`
</ParamField>

<ParamField query="type" type="string" optional>
  Filter by card type: `virtual`, `physical`
</ParamField>

<ParamField query="currency" type="string" optional>
  Filter by currency: `USD`, `EUR`, `GBP`
</ParamField>

## Response Fields

<ResponseField name="cards" type="array" required>
  Array of card objects
  <Expandable>
    <ResponseField name="id" type="string" required>
      Unique card identifier
    </ResponseField>
    <ResponseField name="type" type="string" required>
      Card type: `virtual` or `physical`
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Current card status
    </ResponseField>
    <ResponseField name="balance" type="object" required>
      Current card balance information
    </ResponseField>
    <ResponseField name="limits" type="object" required>
      Card spending limits and remaining amounts
    </ResponseField>
  </Expandable>
</ResponseField>

## Card Status Values

| Status | Description |
|--------|-------------|
| `active` | Card is active and can be used for transactions |
| `frozen` | Card is temporarily frozen by user or system |
| `expired` | Card has passed its expiration date |
| `cancelled` | Card has been permanently cancelled |
| `pending` | Physical card is being processed/shipped |
