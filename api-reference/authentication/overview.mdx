# Authentication

The Entro Finance API uses API key authentication for secure access to platform resources. All API requests must include valid authentication credentials.

## Authentication Methods

### API Key Authentication
API keys provide secure access to your account resources. Include your API key in the Authorization header of every request.

```bash
Authorization: Bearer your_api_key_here
```

### OAuth 2.0
For applications requiring user consent flows, OAuth 2.0 provides secure delegated access to user resources.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST   | /api/auth/keys | Generate new API key |
| GET    | /api/auth/keys | List all API keys |
| DELETE | /api/auth/keys/{keyId} | Revoke API key |
| POST   | /api/auth/oauth/token | OAuth token exchange |

## Base URL

```
https://api.entrofinance.com
```

## Security Best Practices

- Store API keys securely using environment variables
- Rotate keys regularly (recommended: every 90 days)  
- Use minimal required permissions for each key
- Monitor API key usage for suspicious activity
- Implement IP whitelisting for production environments
